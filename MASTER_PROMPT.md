GRG Playscapes Lumber Inventory App — Master Prompt Version 3.0 | Internal Use Only 1. PROJECT OVERVIEW Build an internal-use web app for GRG Playscapes to manage and track lumber inventory across two locations: the Farm and MKE . The app is used by a small team of 5–8 users, with 2–4 active simultaneously. It must function on both mobile devices and desktop computers. This is not a public-facing app. Access is restricted exclusively to authorized GRG Playscapes team members with @grgplayscapes.com Google accounts. 2. TECH STACK Frontend: React (Vite) UI Framework: Tailwind CSS + shadcn/ui Backend &amp; Database: Firebase (Firestore) Authentication: Firebase Auth — Google Sign-In restricted to @grgplayscapes.com domain only Cloud Functions: Firebase Cloud Functions (email triggers only) Email: SendGrid connected to Firebase Cloud Functions, all outgoing mail via sales@grgplayscapes.com Hosting: Firebase Hosting 3. AUTHENTICATION &amp; USER SYSTEM Google Login Firebase Auth with Google Sign-In Restricted to @grgplayscapes.com domain only Users not in the authorized user list cannot access the app Session persists indefinitely — no automatic logout User Roles &amp; Hierarchy Super Admin — John LaPointe (lapointe@grgplayscapes.com) Highest permission level Only person who can demote or remove Dave Full access to all subpages including Admin Control Access to soft-delete restore center Admin — Dave (dave@grgplayscapes.com) Full control of all app features Access to Admin Control and restore center Can be modified only by John Admin — All other users Full app access Protected by soft-delete safety net Soft Delete System Nothing is ever hard-deleted immediately All deletions move to a hidden trash state for 30 days Only John and Dave can view, restore, or permanently delete trashed items All deletion events logged in Audit Log Role-Based Access (Future-Ready) Permission restriction architecture built in but all restrictions off by default John and Dave can toggle restrictions per role from Admin Control without code changes 5. DATA STRUCTURE SKU Data (53 items) SKU — item name/identifier Category — CDR (Cedar), CT (Cedar Tone), GT (Green Treated) Farm Par Level — minimum desired quantity at Farm MKE Par Level — minimum desired quantity at MKE Status — active or discontinued Notes — optional free text Discontinued SKUs Remain visible in inventory until quantity reaches zero Hidden from Par Reports Available for product builds until depleted Auto-deleted (soft delete, logged) when quantity hits zero Location Inventory Every SKU has an independent quantity at each location Par levels, transfers, and reconciliation reports track per location independently Transfer Tracking Transfers are recorded and tracked through requested → shipped → received stages When a transfer is shipped, affected SKUs show an orange &quot;in transit&quot; indicator on the inventory page Inventory quantities are NOT reduced until the transfer is marked received On receive, quantities move from the &quot;from&quot; location to the &quot;to&quot; location Auto-Generated Record Numbers (reset annually) Quote Requests: QR#2026-0001 Transfers: T#2026-0001 Reconciliation Reports: RC#2026-0001 Resets to 0001 every January 1st 6.
